영상 동기화는 어떻게 이루어지나요?

편집 과정에서 상대적 위치, 크기, 타임스탬프, 볼륨 등의 필수 정보(RI: Required Information)를 저장하여 동기화를 구현합니다. 이 데이터는 서버에 업로드되며, 작동 중인 앱이나 플러그인이 해당 정보를 불러와 여러 창을 정확하게 동기화할 수 있도록 합니다.

이 솔루션은 왜 저작권 침해를 피할 수 있나요?

첫째, 당사의 솔루션은 어떤 콘텐츠도 다운로드, 임베드, 수정 또는 결합하지 않습니다.
단순히 수동 조작으로도 가능한 방식으로 여러 개의 창을 동기화하여 재생할 뿐이며, 이는 사용자 단에서 이루어지는 동기화(User-side synchronizer)로 분류됩니다.

둘째, 영상 플랫폼과 계약이 체결된 경우, 해당 플랫폼은 자사 서비스에 대한 권리를 보유하고 있으며, 콘텐츠를 업로드한 IP 소유자에게 사용 허락을 요청할 수 있습니다. 이들이 당사 솔루션에 참여할 경우, 콘텐츠의 조회수가 급증하며 수익 증가로 이어질 수 있습니다. 다만, 참여 여부는 각자의 선택이며, 사용을 원하지 않을 경우 콘텐츠는 전혀 활용되지 않습니다. 따라서 어떤 형태의 무단 사용도 없으며, 법적 문제가 발생하지 않습니다.

셋째, 계약되지 않은 영상 플랫폼의 콘텐츠가 사용되는 경우에도, 각 플랫폼의 서비스 이용약관(ToS) 및 UI 표현을 철저히 준수합니다. 원본 UI는 전혀 변경되지 않으며, 모든 광고 역시 정상적으로 노출됩니다. 계약된 솔루션과 달리 완전한 시뮬레이션은 불가능하지만, 창 인터페이스는 절대 잘려 나가지 않고 전체 화면으로 그대로 표시됩니다.

설명란에 간단한 코드 몇 글자만 추가해도 동기화가 가능한 이유는?

당사의 최종 솔루션에서는 설명란에 별도의 코드를 입력하지 않아도 됩니다. 사용자가 관련 썸네일을 클릭하면, 당사 시스템은 해당 영상의 UI를 서버에 저장된 UI 리스트와 자동으로 비교합니다. 일치하는 경우, 미리 저장된 필수 정보(RI: Required Information)를 활용하여 자동으로 멀티 윈도우 동기화를 수행합니다.

특허에 등록된 주요 구현 방식은 다음과 같습니다:
a) PC 사용자는 플러그인, 모바일 사용자는 앱(Android/iOS)을 한 번 설치하면 됩니다. 이 프로그램은 영상 링크를 인식하고, 해당 영상과 매칭되는 데이터 패키지를 찾아냅니다. 이 패키지는 편집 과정 종료 시 자동으로 서버에 업로드되며, 매우 적은 용량의 데이터로 시각적 동기화가 이루어집니다.
b) 유튜브와 같은 영상 플랫폼이 본 솔루션을 채택하는 경우, 모든 기능을 플랫폼에 직접 내장할 수 있습니다. 이 경우 별도의 소프트웨어 설치나 설명란 코드 입력 없이도 완전한 동기화 기능을 제공합니다.

이 솔루션은 스마트폰에서도 사용할 수 있나요?

In our final solution, there would be no need for adding a code into the description box. When a relevant thumbnail is pressed, our solution will check the UI against our UI list on our server. If there is a match, the required information (RI) will be used synchronize windows. 
Our patent has multiple approaches to solving this problem in our patent.

a) A plugin for computer users and an App for Android or IOS have to be installed once. It will identify the link and find the matching data package. This package is automatically uploaded at the end of every editing process, to our server. This very small data is used by the plugin/App to make the illusion.
b) If our solution is adopted by a video platform, such as YouTube, all necessary functions can be embedded. (No need for S/W installation and adding text code into a description box)

이 솔루션은 추가적인 인터넷 사용량 문제를 일으키나요?

추가적인 인터넷 사용량이 발생하지만, 그 양은 매우 적습니다. 반응 영상의 크기는 원본 영상에 비해 훨씬 작습니다(보통 원본 영상의 약 1/4 크기). 작은 영상의 경우, 해상도를 낮추어도 영상의 선명도에 큰 영향을 미치지 않습니다. 추가적인 데이터 사용량은 약 10%에서 20% 정도로, 대부분의 선진국에서는 큰 문제가 되지 않습니다.

왜 유튜브는 이 솔루션을 구현하지 않았나요?

구글이나 유튜브는 이 아이디어를 구현하지 않은 이유는 해당 기술이 특허로 보호되고 있기 때문입니다. VirtuEDIT Inc.는 이 기술에 대해 미국과 한국에서 특허를 보유하고 있어, 이 기술이 이전에는 존재하지 않았음을 입증합니다. 따라서 구글이나 다른 대기업들은 VirtuEDIT Inc.의 라이선스를 받지 않고는 이 기술을 사용할 수 없으며, 특허 침해를 피하기 위해서는 라이선스 계약을 체결해야 합니다.

VirtuEDIT의 장기적 목표는 무엇인가요?

우리의 목표는 저작권이 있는 콘텐츠를 공정하고 합법적인 방법으로 사용하는 방법을 만드는 것입니다. 우리의 솔루션은 업계를 효율화하여 분쟁과 손실된 수익을 최소화할 것입니다. 콘텐츠의 가능성과 품질은 크게 향상될 것입니다.

OUR VISION

PROBLEM

구시대적인 저작권 시스템

주요제작사와 방송사가 콘텐츠 시장을 독점하던 과거에는 방송사가 지적재산권을 이용하기 위해 저작권자에게 높은 사용료를 지불하며 공정한 거래가 이루어졌습니다.
그러나 접근성이 높아진 영상 제작 환경으로 인해 개인크리에이터들 이미디어 시장을 장악한 지금, 기존의 IP 활용 방식은 시스템적으로 적용할 수 없게 되었습니다.

일반인이 정당하게 IP를 이용하는게 사실상 불가능하기에, 현재 IP 이용은 한 쪽이 불이익을 보는 방식으로 이루어집니다.  이에 맞춰 새로운 시대에 적합한 공정한 솔루션을 찾아야 한다는 목소리가 높아지고 있습니다.

SOLUTION

모두에게 혜택을 주는 가상편집과 멀티윈도우 시스템

가상편집과 멀티윈도우 시스템은 다수의 윈도우 창을 하나처럼 보이도록 재생을 조절해, 다양한 콘텐츠를 하나의 화면에서 통합적으로 즐길 수 있도록 돕습니다.
크리에이터의 영상과 타 저작권 IP 영상이 별도의 창에서 재생되기 때문에, 각각 독립적인 조회수로 인정되어 정확하고 공정한 수익분배가 가능합니다. 더 이상 타인의 지적 재산을 무단으로 사용하거나 침해할 필요가 없습니다.

플랫폼에 공유되는 영상 콘텐츠는 주된 목표로 조회수 증가를 삼기에, 본 솔루션은 저작권자에게 도움을 주게 될 뿐인 것입니다.

VISION

새로운 영상 생태계와 미디어 시장 

저작권 문제 해결을 통해 콘텐츠의 자유로운 활용이 가능해져, 더욱 풍부하고 다양한 창작물들이 탄생할 수 있습니다. 또한 모든 참여자에게 공정한 보상이 돌아가도록 함으로써, 창작 활동의 지속가능성을 확보하고자 합니다.

창작의 자유와 저작권의 보호를 통해 한 단계 진화한 영상 생태계를 만들어 새로운 미디어 르네상스가 올 수 있을 것이라 확신합니다.
